# AGENTS Rules Maintenance

## 目标

本文件定义 `AGENTS.md` 与工作流入口文档的维护原则，避免规则再次膨胀、重叠或漂移。

## 维护原则

- 一条规则只表达一个核心约束。
- 规则之间不得互相冲突、重复或覆盖。
- 优先保留和当前仓库、当前任务直接相关的规则。
- 不要在根入口中堆放通用 LLM 常识或泛化的软件工程口号。
- 不要把环境安装、依赖安装这类 README 内容重复写进 `AGENTS.md`。
- 新增或修改规则后，建议开启新对话再使用，避免历史上下文影响新规则。

## 分层原则

### 应保留在 `AGENTS.md`

- 根入口顺序
- 仓库红线
- 高风险区域
- 强制风格 / 生命周期 / 兼容性约束
- 不能忘的构建 / 运行 / 测试硬约束

### 应放到 `.codestable/attention.md`

- 高频坑
- 启动必读短约束
- 某个阶段里最容易漏掉的短提醒

### 应放到 `workflow-entry.md`

- 任务类型分流
- “下一步读什么 / 跑什么 / 去哪个目录”
- 导航性质的分类和现状判断

### 应放到专项文档

- PR 审查格式与清单
- 提交前验证清单
- 发布说明模板
- 事件 / 遥测摘要模板

## 维护动作建议

- 如果 `AGENTS.md` 出现长段“操作说明”“模板”“分类表”，优先判断是否该下沉。
- 如果新文档是长期稳定入口，先挂到 `workflow-entry.md`，再在 `AGENTS.md` 只保留一条引用。
- 如果某条规则只在单个 feature / issue / roadmap 有意义，不要上升到根入口。
