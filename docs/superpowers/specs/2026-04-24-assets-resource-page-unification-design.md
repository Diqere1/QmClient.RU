# 资源页面与 Workshop 分类统一设计规格

## 概述

本规格定义资源页面后续统一演进的目标结构，使其同时满足以下需求：

1. 资源页面继续作为本地资源管理页，而不是纯远端列表页。
2. 资源分类与 Workshop 数据源保持一致，便于后续扩展新分类。
3. 已下载资源、本地默认资源、旧版兼容资源与远端缩略图缓存需要有清晰职责边界。
4. `gui_cursor`、`arrow`、`strong_weak` 与 `entity_bg` 这些目前行为特殊的资源，需要纳入统一管理模型。

本轮不直接实现功能，只定义可落地的分类模型、迁移规则、路径优先级与 UI 行为，作为后续实现计划的基础。

## 动机

当前资源页面存在 4 类结构性问题：

1. 资源页虽然已经接入 Workshop 远端数据，但当前只覆盖 `HUD`、`entities`、`game`、`emoticons`、`particles`，与后续要增加的分类不兼容。
2. `已下载` 标签尺寸是写死的，和删除按钮的视觉节奏不统一，也无法随文本长度自适应。
3. `gui_cursor`、`arrow`、`strong_weak` 目前属于“固定文件名 + 根目录覆盖”资源，不适合直接复用现有 `assets/<category>` 模型。
4. `entity_bg` 实际消费链路走 `maps` 与 `.map` 文件，而不是普通 `png` 资产，预览与应用方式都明显不同。

## 目标

### 1. 资源页面的总体职责

资源页面必须继续是“本地资源管理页”，而不是纯 Workshop 展示页。

目标行为：

- 已下载项表示用户本地真实可用的资源。
- 未下载项表示来自 Workshop 的可下载条目。
- 远端条目只缓存 `webp` 缩略图用于预览，不把缩略图当作实际资源。
- 删除动作只删除本地安装资源，不删除远端元数据来源。

### 2. 分类统一

本期纳入统一设计的分类如下：

- `entities`
- `game`
- `hud`
- `emoticons`
- `particles`
- `gui_cursor`
- `arrow`
- `strong_weak`
- `entity_bg`

本期排除：

- `audio pack`

`audio pack` 仅作为后续扩展项记录，不进入本期实现计划。

### 3. 分类类型模型

资源页中的分类必须分为 3 类，而不是强行统一为一种目录规则。

#### 3.1 目录型分类

适用于：

- `entities`
- `game`
- `hud`
- `emoticons`
- `particles`

这类资源继续沿用 `DDNet/assets/<category>` 管理。

#### 3.2 单文件集合型分类

适用于：

- `gui_cursor`
- `arrow`
- `strong_weak`

这类资源不再被视为“根目录固定文件名覆盖”，而是作为有名字的资源集合管理。

目标行为：

- 资源页中可以存在多个命名资源。
- 运行时根据当前选中的资源，从 `assets/<category>` 中加载。
- 文件名允许自定义，不再要求只能叫固定名。

英文分类名固定为：

- `gui_cursor`
- `arrow`
- `strong_weak`

#### 3.3 地图型分类

适用于：

- `entity_bg`

这类资源是特殊项，实际消费对象是 `.map` 文件，并通过 `cl_background_entities` 等已有 DDNet 链路生效。

目标行为：

- 资源页统一管理 `entity_bg` 的列表、选择、预览与删除。
- `.map` 有预览图时显示预览，没有时显示默认占位图标。
- 资源页需要接入 DDNet 现有“实体层背景图”设置链路，而不是另起一套孤立配置。

英文分类名固定为：

- `entity_bg`

### 4. Workshop 与分类映射

资源页后续必须以“分类注册表”的方式管理 Workshop 接入，而不是继续按 tab 逐个硬编码分支。

每个分类至少需要定义：

- 分类 id
- 英文目录名
- 本地安装目录
- 是否接入 Workshop
- Workshop 分类名或匹配规则
- 预览类型
- 应用逻辑
- 是否支持删除

无远端源的分类必须保留分类入口，并明确标注：

- `仅本地资源`

### 5. 缩略图缓存

Workshop 远端缩略图缓存继续使用现有路径：

- `DDNet/qmclient/workshop/thumbs`

缓存文件格式继续使用：

- `webp`

目标行为：

- 远端预览缩略图统一缓存到 `qmclient/workshop/thumbs/*.webp`
- 本地真实资源与远端 `webp` 缩略图不能混用
- 删除本地资源时，不强制删除对应的远端缩略图缓存

### 6. default 资源规则

每个资源分类必须始终保留一个 `default` 资源，并固定排在列表第一位。

目标行为：

- `default` 始终存在。
- `default` 不可删除。
- 当选中资源丢失、迁移失败或配置异常时，系统自动回退到 `default`。
- 资源页不允许出现“当前分类没有任何可用资源”的状态，避免空白资源导致程序异常。

注意：

- 虽然 DDNet 内置默认文件在不同模块中的原始文件名未必统一，但资源页对外的管理语义统一为 `default`。

### 7. 旧资源兼容与迁移

对于 `gui_cursor`、`arrow`、`strong_weak`，升级后的兼容迁移规则必须固定为以下顺序：

1. 先为目标分类补齐 `default`
2. 将“当前正在使用的资源”与 `default` 对比
3. 如果当前使用的本来就是 `default`
   - 不做迁移
4. 如果当前使用的不是 `default`
   - 将旧资源导入为 `legacy`
   - 复制到对应资源分类目录
   - 自动把当前应用切换到该 `legacy` 资源

迁移命名规则：

- 首选：`legacy`
- 若重名：使用稳定递增名，例如 `legacy_2`、`legacy_3`

### 8. 运行时加载优先级

对于 `gui_cursor`、`arrow`、`strong_weak`，升级后的运行时加载优先级必须调整为：

1. 先查找 `assets/<category>` 中当前选中的资源
2. 再兼容旧路径：
   - `DDNet/gui_cursor.png`
   - `DDNet/arrow.png`
   - `DDNet/strong_weak.png`
3. 最后回退到客户端内置 `data/` 默认资源

目标效果：

- 资源页可以统一管理多个候选资源。
- 老用户旧路径里的固定文件不会立刻失效。
- 即使新资源丢失，也能稳定回退到内置默认资源。

### 9. entity_bg 的特殊行为

`entity_bg` 在设计上必须被视为“地图型资源”，而不是普通图片资源。

目标行为：

- 本地实体层背景图资源与 `maps` 链路兼容。
- 资源页中可以预览、选择、删除并应用 `entity_bg`。
- 对 `.map`：
  - 有预览图则显示预览图
  - 无预览图则显示默认图标
- 资源页选择后需要接入 DDNet 页面已有的实体层背景图逻辑。
- 需要增加一个用于快速查看的预览开关，使用户可以直接预览 `.map` 地图效果。

本期只定义行为目标，不强制规定最终预览实现是地图编辑器预览、背景组件预览还是专门的轻量预览器。

### 10. 资源页面 UI

资源页面卡片头部的视觉规则需要统一。

目标行为：

- `已下载` 标签高度与删除按钮一致。
- 标签宽度根据文本长度自适应。
- 标签保留最小宽度，但不能像当前一样写死过长。
- 标题区域根据剩余宽度自适应。
- `default` 项的视觉样式与普通项一致，但删除按钮禁用或不显示。

## 架构设计

### 1. 分类注册表

后续实现应引入一个分类注册表，作为资源页和 Workshop 的唯一配置源。

注册表需要回答以下问题：

- 该分类属于目录型、单文件集合型还是地图型？
- 本地安装路径是什么？
- 远端 Workshop 是否接入？
- 远端分类如何匹配？
- 本地扫描规则是什么？
- 应用到客户端时应该写入哪个配置或调用哪个刷新逻辑？
- 预览逻辑使用哪种 renderer？

### 2. 本地资源与远端预览的分层

资源页的数据模型需要显式分层：

1. 本地真实资源
2. 远端可下载资源
3. 远端缩略图缓存

要求：

- 本地真实资源是“可应用”的对象。
- 远端条目是“可下载”的对象。
- 缩略图缓存只是“展示加速”的对象。
- 这三者不能在语义上混用。

### 3. 迁移作为单独步骤

`gui_cursor`、`arrow`、`strong_weak` 的旧路径兼容迁移不能散落在渲染代码中临时判断，应该有独立的迁移入口。

要求：

- 首次进入新模型时完成检查。
- 迁移结果必须稳定落盘。
- 迁移后资源页和运行时读取到的是同一份资源记录。

### 4. entity_bg 的桥接

`entity_bg` 需要桥接现有 DDNet 背景图逻辑，而不是平行造轮子。

要求：

- 资源页负责资源管理体验。
- 现有 DDNet 设置负责最终背景图消费。
- 两者之间通过统一的配置值与刷新入口衔接。

## 成功标准

1. 资源页面可统一管理：
- `gui_cursor`
- `arrow`
- `strong_weak`
- `entity_bg`

2. `gui_cursor`、`arrow`、`strong_weak` 支持：
- `assets/<category>` 优先
- 旧根目录兼容
- 内置默认回退

3. 每个分类都存在 `default`，且始终可回退，不会出现空白资源态。

4. 旧根目录资源可按 `legacy` 规则平滑迁移。

5. `entity_bg` 可在资源页中被统一管理，并能接入现有 DDNet 实体层背景图使用链路。

6. 资源页面卡片头部中：
- `已下载` 标签高度与删除按钮一致
- 标签宽度自适应
- 标题与状态区不再互相挤压

## 风险与非目标

### 风险

1. `gui_cursor`、`arrow`、`strong_weak` 的旧路径兼容如果处理不好，会让老用户现有覆盖资源失效。
2. `entity_bg` 涉及 `.map` 与现有 DDNet 设置链路，预览与应用很容易混在一起。
3. 资源页继续增加分类后，如果没有分类注册表，代码复杂度会快速上升。

### 非目标

1. 本期不把 `audio pack` 纳入资源页面统一管理。
2. 本期不要求一次性重构所有资源消费逻辑为统一目录结构。
3. 本期不要求删除旧根目录兼容能力。
