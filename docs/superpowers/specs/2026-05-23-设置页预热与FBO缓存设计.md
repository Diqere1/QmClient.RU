# 设置页预热与 FBO 缓存设计

> **文档已过时** — 本文档内容不再反映当前代码状态，仅供参考。

## 背景

当前设置页性能优化经历过一版 `CSectionLoader` / compact / deferred 设计，但真实 UI 验收暴露出两个问题：

1. 可见的 compact 摘要文本会从“摘要文本”跳变到“完整控件”，用户感知是卡顿和闪烁，不是优化。
2. TClient 一类页面使用大量本帧局部 lambda，长期保存回调会导致悬垂引用和崩溃风险。

本设计替代旧文档中“compact 标题 + 摘要文本”的方案。新的目标是：**Settings-wide 预热 + 真实 FBO 缓存 + 交互层 immediate-mode**。预热负责冷启动成本，FBO 负责稳态重复绘制成本，任何设置页都不允许用可见摘要文本作为替代渲染。

## 设计目标

1. 设置页预热覆盖整个 Settings 系统，不只覆盖 TClient。
2. 控制页、TClient、QmClient、Graphics、Sound、DDNet 等所有设置页都不能出现“摘要文本 -> 完整控件”的可见跳变。
3. 打开设置页前或打开瞬间，预热真实设置内容需要的字体、文本容器、测量结果和首屏缓存。
4. 设置页稳态下，未 dirty 的 section 使用 FBO blit，减少重复文本布局、控件背景绘制和 draw call。
5. 交互控件仍使用 immediate-mode 路径处理输入、hover、active、文本光标、下拉菜单和颜色选择器。
6. 后端不支持 FBO 时，fallback 必须是完整真实控件渲染，不能回退到摘要文本。

## 非目标

1. 不缓存整张 Settings 页面。整页缓存会在滚动、局部配置变化和交互控件状态变化时大面积失效。
2. 不把控件交互变成“贴图点击”。FBO 缓存只降低静态视觉层成本，不替代 DDNet 现有 immediate-mode UI 语义。
3. 不在本设计中引入新的设置页视觉样式。
4. 不依赖旧 spec 中提到但当前代码未公开提供的 `IGraphics::RenderToTexture`。需要先实现真实 graphics API。

## 当前代码事实

1. `src/engine/graphics.h` 当前没有公开的 `RenderToTexture` / render target API。
2. FBO 相关能力只存在于底层图形后端概念中，例如 OpenGL/Vulkan framebuffer，但没有暴露为客户端 UI 可用接口。
3. `src/game/client/components/menus_settings_controls.cpp` 当前有 `RenderDeferredSummaryBlock()`，这是控制页 compact 摘要路径，必须移除或替换为真实渲染/缓存路径。
4. `CSectionLoader` 已经用于部分设置页实验，并且需要保持“不跨帧持有本帧回调”的约束。

## 推荐方案

采用方案 B：

**Settings-wide 预热 + section 静态层 FBO 缓存 + 交互层 immediate-mode。**

原因：

1. 预热解决首次打开的 glyph rasterization、文本容器创建、测量和首屏准备成本。
2. FBO 解决打开后的稳态重复绘制成本。
3. section 粒度缓存比整页缓存更容易控制 dirty 范围和内存。
4. 交互层保留 immediate-mode，可以避免 hover、输入框、dropdown、color picker 被缓存成死图。

## 总体架构

新增一个 Settings 级别的调度层：

```cpp
class CSettingsWarmupScheduler;
class CSettingsSectionCacheManager;
```

职责拆分：

| 模块 | 职责 |
|------|------|
| `CSettingsWarmupScheduler` | 管理所有设置页的预热队列、每帧预算、会话优先级 |
| `CSettingsSectionCacheManager` | 管理 section 的 FBO 纹理、dirty 状态、尺寸、内存预算 |
| `CSectionLoader` | 管理 section 状态、高度、视口优先级和本帧渲染调度 |
| 各设置页渲染函数 | 注册 section 描述，提供真实 measure/render/prewarm 回调 |
| graphics API | 提供 render target 创建、绑定、结束和 blit 能力 |

长期保存的数据：

```text
section id
page id
cached height
state
dirty/hash
FBO texture/render target handle
cached rect/size
session priority
```

每帧提供的数据：

```text
measure callback
render callback
prewarm callback
viewport
scroll offset
current UI rect
current input/hover/active state
```

回调不能跨帧长期保存。

## 状态模型

旧状态名中的 `COMPACT` 容易误导。新设计建议改为：

```cpp
enum class ESettingsSectionState : uint8_t
{
	UNINITIALIZED,
	MEASURED,
	WARMED,
	CACHED,
	FULL_INTERACTIVE,
};
```

含义：

| 状态 | 含义 |
|------|------|
| `UNINITIALIZED` | 未测量，未预热 |
| `MEASURED` | 已得到真实布局高度 |
| `WARMED` | 已预热真实内容所需字体/文本容器 |
| `CACHED` | 静态视觉层已有有效 FBO |
| `FULL_INTERACTIVE` | 本帧需要完整 immediate-mode 渲染 |

如果为了降低改动保留 `COMPACT` 枚举名，则文档和代码注释必须明确：`COMPACT` 不能渲染可见摘要文本，只能表示“尚未生成 FBO 时的真实等价路径或缓存 blit 路径”。

## 预热层设计

### 范围

预热覆盖所有 Settings 页面：

```text
General
Player
Tee / Tee7
Appearance
Graphics
Sound
DDNet
Custom assets
Controls
TClient
QmClient
TClient Profiles
```

预热不属于 TClient 私有逻辑，应从 `RenderSettings()` / `CMenus` / `CGameClient` 的 Settings 级别调度。

### 优先级

每帧预热预算默认 2-4ms，按以下顺序消费：

1. 上次会话打开的 Settings page 和 scroll 附近 section。
2. 当前正在打开或刚切换到的 Settings page 首屏 section。
3. Controls、TClient、QmClient 等文本和控件密集页。
4. 其它 Settings 页按空闲预算逐步预热。

### 预热内容

预热必须针对真实内容：

1. 测量 section 高度。
2. 触发真实 label 文本的 glyph rasterization。
3. 创建或刷新 text container。
4. 在 render target 能力可用时，为首屏 section 生成 FBO。

预热不能向主 framebuffer 显示摘要文本。若需要离屏渲染，则进入 FBO 或内部文本预热路径。

### 预算规则

```text
每帧预热开始
  取最高优先级 page/section
  measure
  prewarm text/glyph
  如 FBO 支持且 section 可缓存，生成/刷新 section cache
  消耗超过预算则停止，下一帧继续
```

预热过程必须可中断。用户立即打开 Settings 时，应使用已经完成的预热结果，未完成部分走真实 immediate-mode 渲染，不显示摘要。

## FBO 缓存层设计

### Graphics API

需要新增最小 render target 接口：

```cpp
struct SRenderTargetHandle;

virtual SRenderTargetHandle CreateRenderTarget(int Width, int Height) = 0;
virtual void DestroyRenderTarget(SRenderTargetHandle *pTarget) = 0;
virtual bool BeginRenderTarget(SRenderTargetHandle Target, ColorRGBA ClearColor) = 0;
virtual void EndRenderTarget() = 0;
virtual void DrawRenderTarget(SRenderTargetHandle Target, const CUIRect &Dst) = 0;
virtual bool IsRenderTargetSupported() const = 0;
```

实际命名可按 DDNet 图形层风格调整，但语义要完整：

1. 创建 RGBA8 render target。
2. 绑定离屏 target 并清空。
3. 恢复主 framebuffer。
4. 将 target 作为纹理绘制到 UI rect。
5. 后端不支持时可查询并安全 fallback。

至少需要覆盖：

1. OpenGL backend。
2. Vulkan backend。
3. null backend 返回不支持或 no-op。

### 缓存粒度

缓存粒度为 section，不缓存整页。

每个 section 持有：

```cpp
struct SSettingsSectionCache
{
	SRenderTargetHandle m_Target;
	CUIRect m_CachedRect;
	int m_WidthPx;
	int m_HeightPx;
	uint64_t m_ConfigHash;
	uint64_t m_StyleHash;
	bool m_bValid;
	bool m_bStaticLayerOnly;
};
```

### 缓存内容

初版只缓存静态视觉层：

1. section 背景板。
2. 标题。
3. 静态说明文本。
4. 非 active 控件的静态外观。
5. 不依赖鼠标 hover 的分隔线、提示区域、静态 preview。

继续 immediate-mode 的内容：

1. hover/active 状态控件。
2. 正在拖动的 slider。
3. 输入框文字、光标、选区。
4. dropdown popup。
5. color picker popup。
6. 任何本帧需要处理输入或改变布局的控件。

这意味着 section 渲染拆成两层：

```text
RenderStaticLayer()
RenderInteractiveLayer()
```

短期可以先让 `RenderStaticLayer()` 复用现有 full render 的非交互部分；无法拆分的 section 标记为不可缓存，走完整 immediate-mode。

### dirty 规则

以下情况必须使缓存失效：

1. section 依赖的 config 变化。
2. 语言变化。
3. 字体变化。
4. UI scale / window size / HiDPI scale 变化。
5. section 宽度或高度变化。
6. 主题、颜色、透明度变化。
7. section 内存在 active 控件。
8. section 内 overlay 打开，例如 dropdown、color picker。
9. render target 后端重建，例如 graphics device reset。

缓存失效后，本帧 fallback 为真实 full render，并按预算决定是否刷新 FBO。

## 控制页整改

Controls 页不能继续使用 `RenderDeferredSummaryBlock()` 作为可见优化。

整改方式：

1. 删除 Controls 页面 deferred summary 文本路径。
2. Controls 的 Mouse、Controller、Movement、Weapon、Voting、Chat、Dummy、Miscellaneous、Custom 都注册为 settings section。
3. 首屏 section 优先预热。
4. 稳态静态层进入 FBO。
5. bind 列表中的实际按键绑定控件保留 immediate-mode 交互层。

验收时必须专门检查 Controls 页：切到 Controls 后不出现摘要块，不出现后续替换成完整 bind 列表的视觉跳变。

## 兼容与回退

新增配置建议：

```cpp
qm_settings_prewarm = 1
qm_settings_fbo_cache = 1
qm_settings_fbo_cache_debug = 0
```

旧配置 `qm_ui_prewarm` 属于上一版“启动时预热上次打开的设置页文本”方案。它和本方案的产品意图重叠，但实现目标不同；本方案删除该旧配置，只保留 `qm_settings_prewarm` 作为 Settings-wide 预热开关。

回退规则：

1. `qm_settings_prewarm=0`：不预热，但打开后仍完整真实渲染。
2. `qm_settings_fbo_cache=0`：不使用 FBO，所有 section 走真实 immediate-mode。
3. 后端 `IsRenderTargetSupported()==false`：自动禁用 FBO 缓存，不能显示摘要文本。
4. debug 模式可显示 section cache hit/miss、dirty 原因和 render time。

## Profiling

需要记录：

```text
settings_warmup_page
settings_warmup_section
settings_warmup_elapsed_ms
settings_fbo_create_ms
settings_fbo_blit_count
settings_fbo_cache_hit
settings_fbo_cache_miss
settings_fbo_dirty_reason
settings_full_render_ms
```

目标不是只看平均值，而是能定位某个页面、某个 section、某个 dirty 原因。

## 测试策略

### 单元测试

1. `CSectionLoader` 不跨帧保存 callbacks。
2. section 状态按 `MEASURED/WARMED/CACHED/FULL_INTERACTIVE` 推进。
3. dirty config 变化后缓存失效。
4. 后端不支持 render target 时 fallback 到 full render。
5. 控制页不能注册 summary-text section。

### 集成测试

1. 模拟 `CSettingsWarmupScheduler` 多页预热队列，验证预算不会被单页耗尽。
2. 模拟上次会话 page/scroll，验证优先预热命中 section。
3. 模拟语言切换和窗口 resize，验证所有 FBO cache invalidated。

### 手动验收

1. 首次打开 Settings，不出现摘要文本。
2. 打开 Controls，不出现 summary block，也不出现“摘要 -> bind 列表”的跳变。
3. 打开 TClient Settings，Pet、Background Draw、StatusBar 等下半部分内容可滚动可见。
4. 切换 TClient 子 tab 不崩溃。
5. 开启 profiling 后，稳态滚动命中 FBO cache，full render 次数下降。
6. 禁用 FBO 后，页面仍完整可交互，只是性能回退。

## 实施顺序

1. 清理所有 Settings 可见 compact/deferred summary 路径，尤其 Controls。
2. 把 Settings 页面拆成 section registry，不跨帧保存本帧 callbacks。
3. 实现 Settings-wide warmup scheduler，先做 glyph/text/measure 预热。
4. 增加 render target graphics API，并实现 OpenGL/Vulkan/null backend。
5. 为 section 接入 FBO 静态层缓存。
6. 增加 dirty 规则和 profiling。
7. 对 Controls、TClient、QmClient 做重点验收。

## 验收标准

1. 全 Settings 范围内没有可见 compact 摘要文本优化。
2. Controls 页无 `RenderDeferredSummaryBlock()` 可见路径。
3. 预热范围覆盖所有 Settings 页面，并按会话/当前页优先级执行。
4. 后端支持 FBO 时，稳态未 dirty section 使用 FBO blit。
5. 后端不支持或配置关闭 FBO 时，fallback 为完整真实控件渲染。
6. TClient 子 tab 切换不崩溃。
7. TClient Settings 下半部分内容不丢失。
8. 语言切换、窗口 resize、字体变化后缓存正确失效。
9. `run_cxx_tests` 通过。
10. `game-client` 构建通过。

## 旧设计废弃点

以下旧设计点不再成立：

1. `COMPACT` 渲染标题 + 摘要文本。
2. 只预热 TClient/QmClient 页面。
3. 假设当前 `IGraphics` 已有公开 `RenderToTexture`。
4. 全局一次性注册 frame-local callbacks。
5. 用 deferred frames 控制可见摘要块。

后续计划和实现必须以本文档为准。
