# 资源页、实体层背景图与素材编辑器扩展设计规格

## 概述

本规格定义以下 4 条主线的统一设计：

1. `entity_bg` 资源页的目录语义、排序规则与 Workshop 展示模型。
2. 资源页底部的商店资源显示开关与相关文案调整。
3. 材质编辑器对 `strong_weak`、`skin` 与 `audio pack` 的扩展方向。
4. 资源分享能力的边界与后续预研结论。

本规格的目标不是一次性重构整套资源系统，而是在不破坏 DDNet 现有运行时链路的前提下，给当前分支提供一套可分步实现、可验证、可回退的设计。

## 动机

当前分支已经完成一部分资源页统一工作，但仍存在 5 类结构性问题：

1. `entity_bg` 同时存在“本地层级视图”和“Workshop 平铺追加”两套展示模型，导致根页、子目录页与未下载资源的显示语义不一致。
2. `default` 在 `entity_bg` 页中仍然沿用普通排序，无法满足“排在文件夹之后第一个”的使用预期。
3. 资源页的 Workshop 可见性目前只由刷新/数据状态隐式决定，没有单独的“显示开关”。
4. 现有材质编辑器虽然已经支持拼装、着色和导出 PNG，但对 `strong_weak` 的切分不符合需求，对 `skin` 与 `audio pack` 也没有正式入口。
5. “联机中玩家对玩家发送贴图资源”已被提出，但当前服务端不在可控范围内，无法作为本轮交付目标直接承诺。

## 范围

### 本轮纳入实现计划

- `entity_bg` 目录视图与排序规则重构
- 资源页商店资源显示开关
- `Sync Workshop Assets` / `Show Workshop Assets` 文案与国际化
- `strong_weak` 编辑切分完善
- `skin` 编辑模式设计与接入
- `audio pack` 编辑与导出到 `audio/<pack>/...`

### 本轮不纳入实现

- 基于服务端的联机资源分享
- 资源分享的端到端加密
- `audio pack` 的分享型 zip 导出强依赖

### 本轮允许作为扩展项记录

- `audio pack` 的可选 zip 导出
- 资源分享的 `pie menu` 入口与交互原型

## 目标

### 1. `entity_bg` 必须只有一套显示模型

`entity_bg` 页面不能继续混用：

- 本地资源走层级视图
- 未下载 Workshop 资源走平铺追加

本轮目标是将 `entity_bg` 收敛为单一层级视图。

目标行为：

- 根页只显示本地目录、本地根层资源、`assets/entity_bg (Workshop)` 目录入口与 `default`。
- 未下载 Workshop 资源只在 `assets/entity_bg (Workshop)` 目录内展示。
- 进入本地目录后，只显示该目录下的本地资源，不混入其它来源的远端项。
- 搜索状态下允许打平结果，但仍要在视觉和行为上保留 `entity_bg` 专用逻辑，不退回普通 tab 的 combined list 模式。

### 2. `default` 的排序必须独立于普通资源

`entity_bg` 页中的 `default` 不再简单复用通用名字排序。

固定排序规则：

1. `..`
2. 文件夹
3. `default`
4. 普通资源

该规则适用于：

- 根目录
- 子目录
- `assets/entity_bg (Workshop)` 目录

其中 `default` 必须始终存在、不可删除、可被选中为回退项。

### 3. 资源页增加显式的商店资源显示开关

资源页底部新增按钮：

- 中文：`显示商店资源`
- 英文：`Show Workshop Assets`

按钮语义：

- 这是一个显示开关，不是同步按钮。
- 默认开启。
- 关闭时，本地资源仍然完整可见，只隐藏 Workshop 资源和相关远端预览内容。
- 开启时，普通资源页按既有 tab 规则展示 Workshop 项，`entity_bg` 页则按本规格的目录规则展示 Workshop 项。

原有按钮文案统一改为：

- 中文：`同步商店资源`
- 英文：`Sync Workshop Assets`

该按钮只负责刷新远端数据，不承担“显示/隐藏”的职责。

### 4. `entity_bg` 与商店资源开关的联动规则

`entity_bg` 页在开关关闭时：

- 根页不显示 `assets/entity_bg (Workshop)` 目录入口。
- 当前若已经处于 `assets/entity_bg (Workshop)` 目录，界面应安全回退到根目录。
- 仅保留本地目录、本地资源与 `default`。

`entity_bg` 页在开关开启时：

- 根页显示 `assets/entity_bg (Workshop)` 目录入口。
- 未下载 Workshop 资源只在该目录中显示。
- 不允许在根页直接平铺未下载 Workshop 预览。

### 5. `strong_weak` 编辑切分必须符合新验收标准

`strong_weak` 继续走现有材质编辑器与 PNG 导出链路，不单独做一套新编辑器。

但其 slot 切分规则改为：

- 按 3:1 纵向区域切出三份
- 每份按 1:1 小格处理
- 交互仍保持：
  - 左键短按：打开颜色选择器
  - 左键长按/拖动：替换材质来源
  - 右键：恢复当前格子的默认来源

这意味着实现重点在于：

- slot 定义
- 预览高亮
- 导出回写区域

而不是重写整套编辑器交互。

### 6. `skin` 不进入资源页分类，而是新增编辑器模式

`skin` 不作为资源页 tab 新分类实现。

原因：

- 运行时皮肤系统不是单 PNG 分类，而是 `skins7` 的多部件 + 颜色配置模型。
- 若强行挂到资源页，会把“编辑皮肤”和“管理资源文件”两个不同职责混在一起。

本轮目标是：

- 在现有材质编辑器中新增 `skin` 模式
- 提供显式切片表，而不是依赖自动 sprite 推断
- 复用现有画布、拖拽、着色、预览与状态提示
- 导出结果对齐 `skins7` 当前保存格式

### 7. `skin` 的切分必须按参考图定义

参考图明确表达了皮肤编辑所需的切块语义，而不是任意网格。

第一版显式切片表按以下方向定义：

- 身体主块：`96x96`
- 脚主块：`96x96`
- 右侧竖向附加区域：按图中的 `32x32`、`64x32` 片段定义
- 下方小块：按一排 `32x32` 定义

其中“强弱钩”的编辑验收继续以三段 1:1 切块为准，不要求完全照搬右侧示意图的每一块都在第一版上线。

本轮要求的不是“自动识别所有皮肤布局”，而是先把当前 QmClient 实际需要的切分方案显式化、稳定化。

### 8. `skin` 导出必须对齐现有 `skins7` 保存模型

`skin` 编辑结果不能只停留在编辑器预览纹理。

导出目标必须包括：

- 对应贴图素材输出
- `skins/<name>.json` 写回
- 颜色配置与 `custom_colors` 语义对齐

目标行为：

- 编辑后的皮肤可被客户端现有皮肤系统直接识别
- 不引入平行的“编辑器专用皮肤格式”
- 不修改现有 `skins7` 文件格式定义

### 9. `audio pack` 第一版以目录包为主

当前运行时音频包加载规则已经明确支持：

- `audio/<pack>/...`
- `audio/<pack>/audio/...`

因此第一版音频包编辑器的目标不是“先做 zip 包”，而是：

- 可浏览声音槽位
- 可重新映射对应声音文件
- 可试听
- 可导出到 `audio/<pack>/...`
- 导出后客户端可立即在设置页中识别并加载

这部分不进入资源页分类，而是独立编辑入口。

### 10. `audio pack` 的 zip 导出是可选扩展

本轮不将 zip 导出作为主交付承诺。

原因：

- 当前仓库中没有明确的、已接入当前流程的通用 zip 编辑/写包能力可直接复用
- 运行时本身消费的是目录包，而不是 zip 包

因此 zip 导出在规格中仅作为可选增强项：

- 若实现成本低，可将当前 `audio/<pack>` 目录额外打成分享包
- 若成本高，不阻塞第一版音频包编辑功能交付

### 11. 材质编辑器的通用交互要求保持一致

无论是 `strong_weak`、`skin` 还是后续扩展类型，编辑器交互统一保持以下规则：

- 左键短按当前小格：打开 RGBA 颜色选择器
- 左键长按或拖动：开始替换材质
- 右键当前小格：恢复该格默认状态

同时必须修复和维持以下交互质量目标：

- 颜色选择器打开时，背后格子与结构性按钮不能继续响应点击
- 当前被调色的小格必须有稳定可见的高亮反馈
- target 侧重叠区域必须具备可控的命中逻辑，避免总落在默认候选

## 设计

### 1. `entity_bg` 的视图模型

`entity_bg` 继续保留专用数据结构，但不再复用普通资源页的 remote append 思路。

视图分层如下：

1. 本地源列表
2. Workshop 源列表
3. 当前目录过滤器
4. 层级项构建器
5. 卡片渲染

关键设计要求：

- 层级项构建器必须同时知道“显示路径”和“来源类型”
- 不能只靠名字前缀判断 Workshop 语义
- 目录入口与资源卡片都必须由同一轮构建结果产出

### 2. `entity_bg` 的最小来源模型

本轮不强制引入跨全项目的大型注册表重构，但在 `entity_bg` 内部至少应区分：

- `maps` 来源
- `assets/entity_bg` 本地管理目录来源
- `workshop_virtual` 虚拟目录来源

这样可以避免：

- `maps/entity_bg/...` 被误判成 Workshop 目录
- 删除时错误地落到 `assets/entity_bg/...`
- 预览和选择路径与显示路径混淆

### 3. 资源页商店开关的状态模型

商店资源显示开关应为资源页级 UI 状态，并具备以下属性：

- 默认值为开启
- 至少在当前会话内稳定生效
- 切换 tab 后行为可预测

本规格不强制第一版就将该状态持久化到配置文件。

推荐第一版行为：

- 会话级静态状态
- 在所有资源页共用
- `entity_bg` 与普通资源页统一读取

### 4. `skin` 编辑模式的结构

`skin` 模式使用现有编辑器框架，但在以下层面做专用化：

- 资源扫描源：从皮肤相关目录与默认部件获取
- slot 定义：由显式切片表提供
- 导出逻辑：输出到 `skins` 运行时可识别结构
- 预览：右侧组合图继续由现有 composed preview 生成

这意味着第一版不追求“任意皮肤格式都能自动识别”，而是先服务于当前客户端自定义皮肤编辑。

### 5. `audio pack` 编辑器的结构

第一版 `audio pack` 编辑器采用“声音槽位映射器”模型：

- 左侧：声音槽位列表
- 右侧：当前槽位的候选文件、预览与试听
- 导出：将选择结果写到 `audio/<pack>/...`

第一版不要求做波形编辑、裁剪、音量包络等音频编辑功能。

YAGNI 范围：

- 只做文件级重组
- 不做音频内容编辑
- 不做跨包批量同步

### 6. 资源分享的结论

“联机中玩家对玩家分享资源”已确认是有价值的扩展方向，但本轮不实现。

当前结论：

- 入口优先考虑 `pie menu`
- 第一版若未来实现，应优先复用现有 Qm 服务标识体系
- 优先走“客户端发起 + 外部服务中转/协调”，而不是直接改 DDNet 主协议
- 由于服务端当前不在控制范围内，本轮不做实现承诺

## 错误处理

### 1. `entity_bg`

- 若当前目录在关闭商店资源后失效，自动回退到根目录
- 若当前选中资源被删除或来源失效，回退到 `default`
- 若 Workshop 数据为空，不显示空目录占位卡片

### 2. `skin`

- 若某个切片资源缺失，回退到默认皮肤部件
- 若颜色配置无效，回退到默认颜色
- 若导出名非法，不写文件并显示明确错误

### 3. `audio pack`

- 若目标声音文件不存在，不允许写出损坏映射
- 若试听文件不可读，保留当前选择但提示无法试听
- 若导出目录无法创建，显示写入失败并保持编辑状态

## 测试与验证

### 1. `entity_bg`

必须验证：

- 根页只在开关开启时显示 `assets/entity_bg (Workshop)` 入口
- 根页排序为“文件夹 -> default -> 普通资源”
- 进入 `assets/entity_bg (Workshop)` 后才显示未下载 Workshop 资源
- 关闭开关后从该目录安全回退
- 本地 `maps` 资源仍可浏览、应用、删除

### 2. 资源页商店开关

必须验证：

- 默认开启
- 切换后当前 tab 立即生效
- 文案与国际化正确
- `Sync Workshop Assets` 与 `Show Workshop Assets` 职责不混淆

### 3. `strong_weak`

必须验证：

- 三段切分命中正确
- 左键调色、长按拖拽、右键恢复符合预期
- 导出 PNG 后客户端运行时显示正确

### 4. `skin`

必须验证：

- 参考图切分与预览一致
- 小格子命中正确
- 颜色变化有即时视觉反馈
- 导出后皮肤系统可直接加载

### 5. `audio pack`

必须验证：

- 导出到 `audio/<pack>/...` 后设置页可识别该包
- 切换到该包后运行时按映射加载声音
- 默认包与自定义包切换不崩溃

### 6. 构建与自动验证

本轮实现完成前至少需要运行：

- `qmclient_scripts\\cmake-windows.cmd --build build-ninja --target game-client -j 10`
- `qmclient_scripts\\cmake-windows.cmd --build build-ninja --target run_cxx_tests -j 10`

## 非目标

以下内容不属于本规格的实现目标：

- 重写整个资源页架构
- 修改 DDNet 主协议以支持资源传输
- 承诺资源分享的端到端加密
- 对音频内容本身做剪辑、混音或格式转换编辑

## 开放问题结论

### 1. 商店资源默认是否显示

结论：默认显示。

### 2. 音频包第一版是否必须 zip 导出

结论：不是。第一版以 `audio/<pack>/...` 目录导出为主，zip 为可选增强。

### 3. 资源分享是否纳入本轮实现

结论：不纳入。本轮只记录为后续扩展项。
