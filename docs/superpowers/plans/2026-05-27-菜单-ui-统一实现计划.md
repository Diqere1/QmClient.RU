# 菜单 UI 统一实现计划

> **部分内容已过时** — 部分描述不再反映当前代码状态，请以较新的相关文档为准。

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 把当前 QmClient 菜单主路径收口到同一套 menubar 规则和半透明玻璃卡片系统，并补上 skins 预览/FBO cache 的一致性修复。

**架构：** 以 `menus.cpp` 为总入口，抽出统一的菜单卡片配色和布局常量，再让 browser/settings/demo 统一走这套来源。资源页额外处理预览绘制和 settings page runtime cache 的命中条件，避免布局变了但继续复用旧 FBO。

**技术栈：** C++、DDNet/QmClient 菜单系统、QmUi token、settings runtime cache/FBO、Windows Ninja 构建脚本
