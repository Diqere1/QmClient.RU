# 设置页花屏与皮肤预览缓存修复 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复 TClient 设置页 page-cache 花屏/脏帧回归，并补齐 Tee 皮肤列表 preview cache 的后台生成闭环，让缓存既不污染交互帧，又能在跨会话中真正命中。

**Architecture:** 这轮不重写全局 scheduler。先把已经部分落地的 TClient page-cache correctness 链补成可证明的 live 行为，再把 Tee preview cache 拆成两条明确路径：visible row 继续 live `RenderTee(...)` / cache-first 消费，后台 maintenance 独立负责 render-target readback、12-layer artifact 写盘和 manifest 发布。source-skin warmup 与 preview-cache maintenance 必须继续解耦。

**Tech Stack:** C++17, DDNet/QmClient menu/runtime-cache code, GoogleTest, render target/readback APIs, WebP encode/decode helpers, Windows Ninja/MSVC build through `qmclient_scripts/cmake-windows.cmd`

- [ ] **Step 3: 在 `menus_settings.cpp` 新增最小 generation worker 状态，并按帧推进**

```cpp
struct SQueuedSkinPreviewGeneration
{
	SSettingsSkinPreviewCacheKey m_Key;
	std::string m_SkinName;
	int m_Emote = 0;
	int m_NextLayer = 0;
	IGraphics::CRenderTargetReadbackHandle m_Readback;
	std::array<CImageInfo, NUM_SETTINGS_SKIN_PREVIEW_CACHE_LAYERS> m_aResolvedImages;
	std::array<std::string, NUM_SETTINGS_SKIN_PREVIEW_CACHE_LAYERS> m_aTemporaryLayerPaths;
};

static std::deque<SQueuedSkinPreviewGeneration> s_vQueuedPreviewGenerations;
```

维护规则：
- 同 key 不重复排队；
- 只在 `PreviewCacheMaintenanceAllowed` 时推进；
- 每帧最多消耗 1 次 readback / 1 次 preview-cache I/O token。
- readback 是跨帧状态机：render layer -> `BeginRenderTargetReadback(...)` -> 后续帧 `PollRenderTargetReadback(...)` -> `ResolveRenderTargetReadback(...)` -> 写临时 WebP -> 全部 layer 完整后 publish。
- 任何 pending readback 在页面退出、key 失效或 worker 被丢弃时必须 `CancelRenderTargetReadback(...)`，并清理已解析 image / temp path。

- [ ] **Step 4: 用 render target + readback + WebP 写盘实现单 key 生成**

```cpp
ESettingsSkinPreviewGenerationStepResult AdvanceSettingsSkinPreviewCacheGeneration(
	IGraphics *pGraphics,
	IStorage *pStorage,
	CSettingsSkinPreviewCache &Cache,
	SQueuedSkinPreviewGeneration &Job,
	const CSkin *pSkin,
	int Emote)
{
	// One bounded maintenance step only:
	// 1. If no readback is pending, render one preview layer into the reusable target and BeginRenderTargetReadback(...)
	// 2. If readback is pending, PollRenderTargetReadback(...)
	// 3. When ready, ResolveRenderTargetReadback(...), SaveWebP(...) to a temporary layer path, and advance m_NextLayer
	// 4. After all layers are present, fill manifest metadata and Cache.PublishDiskCacheArtifactsAtomically(...)
}
```

实现要求：
- 任一 layer/readback/save 失败都立刻放弃本 key；
- 不写半组正式文件；
- manifest 最后写；
- 成功后可选择 `LoadTexturesFromDisk(Key)` 再 `RememberTextures(...)`，确保后续会话和当前会话都能命中同一套产物。
