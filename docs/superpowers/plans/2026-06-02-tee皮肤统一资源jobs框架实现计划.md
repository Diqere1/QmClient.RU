# Tee 皮肤统一资源 Jobs 框架实现计划

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 把 `设置 -> Tee` 从“可见时才勉强补资源、回看可能重新 loading”的体验，改成 source/live render 正确性基线清晰、派生 cache 角色清晰、回看稳定的资源模型，并把它接到可复用的 settings resource control-plane 上，同时不破坏现有资源页 / Workshop 的成熟预算语义。

**架构：** 先用影子模式补齐 preview artifact key、cache-disabled reason、budget/evict reason 的观测，再把 Tee 列表发布契约拆成 `Skeleton Ready / Resources Settled`。随后先调研并选择合理的 skin loading / preview artifact 路径，而不是预设“单图缓存”或“12 layer 缓存”一定正确；确定路径后再落地 stable residency：`cl_skins_loaded_max` 只限制 pending/loading 源图加载队列，不限制已 loaded source 或 stable preview。最后再把 source/preview 的 eviction 从 count-fuse 迁移到 bytes-driven 主预算，并对齐资源页 / Workshop 的 control-plane 语义。

**技术栈：** C++17, DDNet/QmClient menu and skins components, GoogleTest (`testrunner.exe`), settings resource jobs helpers, WebP preview cache, Windows Ninja/MSVC build through `qmclient_scripts/cmake-windows.cmd`
