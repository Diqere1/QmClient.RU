# 设置页虚拟化 C 方案实现计划

> **文档已过时** — 本文档内容不再反映当前代码状态，仅供参考。

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 一次性完成 specs 中的 Settings-wide C 方案：覆盖 General、Language、Player、Tee、Controls、Graphics、Sound、DDNet、TClient、QmClient、Appearance、Assets、Skins 等设置页，消除首次进入、滚动、切页、切子 tab、资源列表首次访问时的交互帧重活。

**架构：** 做大一统重构，但边界必须清晰：新增 Settings runtime cache/metadata/warmup scheduler 作为统一核心；所有设置页通过统一 page/section/text/resource cache key 注册；资源型页面通过现有 Jobs 池生成 list/image plan，主线程预算合并和上传；FBO 只缓存静态层，交互层实时渲染。禁止 compact/summary 可见文本优化，禁止 Jobs 访问 `CUi`、`TextRender`、`Graphics`。

**技术栈：** C++、DDNet immediate-mode UI、现有 `IJob` Jobs 池、render target/FBO、`GpuUploadLimiter`、GoogleTest/C++ tests、现有 Windows build gate。

#include <algorithm>

static void AddUnique(std::vector<int> &vValues, int Value)
{
	if(std::find(vValues.begin(), vValues.end(), Value) == vValues.end())
		vValues.push_back(Value);
}

SSettingsPageRuntimeRegistry BuildSettingsPageRuntimeRegistry()

	SSettingsPageRuntimeRegistry Registry;
	AddUnique(Registry.m_vPages, CMenus::SETTINGS_LANGUAGE);
