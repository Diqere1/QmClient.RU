# qmclient_scripts 脚本总览

## 目标

本文件统一说明 `qmclient_scripts/` 目录里各类脚本的职责边界、推荐入口和使用场景。

原则只有一条：

- 先看“这类问题属于哪一层”，再决定调用哪个脚本

一句话理解：

- 这份总览不是“脚本清单”，而是“选入口的地图”
- 先分清是门禁、构建、卫生检查，还是某个专用脚本，再跑对应入口
- 避免把单项脚本当成总入口，也避免临时拼一套重复门禁

## 目录分层

### 1. `gate/`

这是仓库级门禁与脚本治理层。

当前规范入口：

| 入口 | 说明 |
|------|------|
| `qmclient_scripts/gate/check_gate.py` | **Python 版仓库级门禁总入口** |
| `checks/strict_build` | 严格构建与静态分析（`check_gate.py` 调度模块） |
	| `qmclient_scripts/gate/check_docs.py` | 治理文档一致性检查（AGENTS/Claude 同步 + 工作流文档） |
	| `qmclient_scripts/gate/tools/refresh_allowlist.py` | 刷新基线白名单 |
| `qmclient_scripts/gate/check-gate-workflow.md` | 门禁工作流文档 |
| `qmclient_scripts/gate/baseline_debt_allowlist.json` | 基线白名单数据 |

适用：

- 跑仓库级 `quick/default/full` 门禁
- 跑严格构建、`/analyze`、clang-tidy、ASan
- 校验 `AGENTS.md` / `CLAUDE.md` / `.ai/` / CI 入口是否一致
- 维护 baseline debt allowlist

### 2. 构建与平台辅助

这类脚本负责“让构建或平台行为成立”，不是门禁总入口。

当前主要脚本：

- `qmclient_scripts/cmake-windows.cmd`
- `qmclient_scripts/darwin_fix_install_names.py`
- `qmclient_scripts/make_lib_openssl.sh`

适用：

- Windows 上通过已注入环境的 `cmd` 链调用 CMake
- macOS 安装名修正
- OpenSSL 相关本地构建辅助

### 3. 代码卫生与内容生成辅助

这类脚本是具体检查项或生成项，不负责仓库级编排。

当前主要脚本：

- `qmclient_scripts/check_config_variables.py`
- `qmclient_scripts/check_header_guards.py`
- `qmclient_scripts/fix_style.py`
- `qmclient_scripts/export_settings_commands_table.py`
- `qmclient_scripts/generate_release_notes.py`

适用：

- 单独检查配置变量使用
- 单独检查头文件 guard
- 单独跑样式检查
- 导出设置命令表
- 生成发布说明草稿

### 4. 其他仓库或服务脚本

这类脚本与门禁主链无直接关系，按各自职责独立存在。

当前可见目录/脚本：

- `qmclient_scripts/languages_qmclient/`
- `qmclient_scripts/qmclient_center_server/`
- `qmclient_scripts/diff_update.py`
- `qmclient_scripts/tw_api.py`
- `qmclient_scripts/update.zsh`

如果任务不属于“门禁 / 构建 / 代码卫生”，通常先从这里判断是不是专用子系统脚本。

## 推荐入口

### 想跑仓库级门禁

先用：

| 命令 | 说明 |
|------|------|
| `python qmclient_scripts/gate/check_gate.py --mode quick` | 源码卫生快速自查（Python 版） |
| `python qmclient_scripts/gate/check_gate.py --mode default` | 日常提交前严格门（Python 版） |
| `python qmclient_scripts/gate/check_gate.py --mode full` | 集中收口（Python 版） |
说明：

- `quick`：源码卫生快速自查
- `default`：日常提交前严格门
- `full`：集中收口

### 想单独跑严格构建与静态分析

先用：

| 命令 | 说明 |
|------|------|
| `python qmclient_scripts/gate/check_gate.py --mode default` | 严格构建与静态分析（Python 版） |
### 想确认脚本 / 文档 / CI 入口有没有漂移

先用：

- `qmclient_scripts/gate/check_docs.py`
- `qmclient_scripts/gate/check_workflow_docs.py`

### 想同步 `AGENTS.md` 和 `CLAUDE.md`

先用：

- `qmclient_scripts/gate/sync_agents_claude.py`
### 想刷新 baseline allowlist

先用：

	- `qmclient_scripts/gate/tools/refresh_allowlist.py`

### 想生成发布说明

先用：

- `qmclient_scripts/generate_release_notes.py`

## 不要这样用

- 不要把 `check_config_variables.py`、`fix_style.py`、`check_header_guards.py` 误当成仓库级总入口
- 不要绕开 `qmclient_scripts/gate/check_gate.py` 自己临时拼一套"等价门禁命令"
- 不要把 `qmclient_scripts/` 根目录当成“全都互相平级”；门禁相关内容统一以 `gate/` 为准
